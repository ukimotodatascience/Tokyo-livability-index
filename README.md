# 東京23区向け 一人暮らし意思決定支援ツール (Tokyo Livability Index)

本プロジェクトは、公的統計やオープンデータを利用して東京23区の住環境・治安・利便性・防災等の指標を多面的に可視化し、一人暮らしを検討している方の最適な居住エリア（区）選定を支援する意思決定ダッシュボードです。

## 特徴
- **ハイブリッドなデータ構成**: APIによるリアルタイムデータ（OpenStreetMap/Overpass API, 警視庁犯罪統計）と、e-Statなどの公的統計マスタを組み合わせた頑健な設計。
- **動的なおすすめ度（スコアリング）**: ユーザー自身が「治安」「家賃」「アクセス」「生活利便性」など各観点の優先度をカスタマイズでき、それに応じたおすすめ順ランキングが作成されます。
- **23区インタラクティブマップ**: 地図（SVG）から視覚的に各区の特徴や位置、スコアの分布を確認できます。
- **候補エリアの比較**: 複数の候補区を横並びで詳細に比較できるテーブル機能を搭載。

## 構成・ファイル構成
- [main.py](main.py): データ取得、前処理、スコアリング、フロントエンド組み込みデータ作成までを一気通貫で行うパイプライン。
- `src/`: データのコレクターモジュール群（`estat_collector.py`, `crime_collector.py`, `osm_collector.py`, `spatial_collector.py`, `area_collector.py`）。
- `data/`:
  - `raw/`: 収集した生CSVファイルや、静的な公的統計（`tokyo_wards_master.csv`）が格納されています。
  - `processed/`: パイプラインにより処理・スコアリングされた最終出力 `tokyo_livability_index.csv` が保存されます。
- [index.html](index.html): ダッシュボードのメインUI。
- `assets/`:
  - `styles.css`: UIスタイルシート。
  - [app.js](assets/app.js): ダッシュボードのロジック・対話性（SVGマップ描画、スコア再計算など）。
  - `embedded-data.js`: パイプラインによって出力された静的組み込みデータ（オフラインでも動作可能）。

## セットアップと起動方法

### 1. 依存関係のインストール
本プロジェクトの依存ライブラリは `requirements.txt` に記載されています。
仮想環境等を利用してインストールしてください。

```bash
pip install -r requirements.txt
```

### 2. データパイプラインの実行（データの最新化）
```bash
python main.py
```
※ e-Stat APIキーが環境変数等に指定されていない場合、自動的にローカルの人口統計フォールバックを使用します。
※ 警視庁の犯罪統計（令和5年度確定値）や、OSMの駅・避難所数をフェッチ・更新します。

### 3. ダッシュボードの起動（ローカルサーバー）
本ダッシュボードはHTML/JS/CSSのフロントエンドアプリです。ローカルで動作確認を行うには、簡易的なHTTPサーバーを立ち上げてください。

```bash
python -m http.server 8000
```
起動後、ブラウザで http://localhost:8000 にアクセスしてください。
